# API Hook测试用例

## 测试目的
验证API Hook功能的正确性，确保能成功拦截和模拟硬件信息

## 测试环境
- 操作系统：Windows 10 x64
- .NET Framework：4.8

## 测试步骤
1. 启动Hook工具
2. 注入目标进程
3. 运行硬件信息测试程序，验证以下硬件信息是否被正确模拟：
   - CPU信息（型号、核心数）
   - 硬盘信息（序列号）
   - MAC地址
   - 主板序列号
4. 测试不同Hook模式（Normal、Capture、Replay）
5. 测试Hook回调函数的异常处理
6. 测试Hook的稳定性（长时间运行）

## 预期结果
- 所有硬件信息能被正确模拟
- 不同Hook模式能正常工作
- Hook回调函数异常不会导致目标进程崩溃
- Hook能长时间稳定运行
